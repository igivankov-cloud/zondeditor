# ZondEditor Regression Checklist (Manual)

Цель: быстрый ручной прогон после изменений в архитектуре состояния/диагностики/экспорта.

## 1) K2M базовый сценарий (open -> edit -> save -> reopen -> export)
1. Открыть K2 GEO (`fixtures/K2_260205A1.GEO` или рабочий файл).
2. Проверить common params (шкала/Fкон/Fмуф/площади) в ленте и статусе.
3. Изменить несколько значений qc/fs (включая один пропуск и одно исправление).
4. Проверить шапку/легенду/нижний статус/протокол ошибок.
5. Сохранить `.zproj`, закрыть, открыть снова.
6. Убедиться, что параметры, вид и статусы совпали после reopen.
7. Экспортировать GEO + GXL + Excel + CREDO ZIP.

## 2) K4M базовый сценарий (probe params + graphs + reopen + export)
1. Открыть K4 GXL/GEO (`fixtures/k4.gxl` или рабочий файл).
2. Проверить параметры зонда (тип, шкала, усилия, площади).
3. Включить графики: убедиться, что шкалы и линии соответствуют данным.
4. Сохранить `.zproj`, закрыть, открыть снова.
5. Проверить, что `geo_kind`, common params, графики и статусы сохранились.
6. Экспортировать GXL + Excel + CREDO ZIP.

## 3) Последовательная загрузка форматов
1. Загрузить K2 -> проверить common params.
2. Загрузить K4 -> проверить, что параметры K2 не «протащились».
3. Снова загрузить K2 -> проверить обратный переход.

## 4) Диагностика и визуальные статусы
1. Создать опыт с >5 нулями подряд в qc или fs.
2. Проверить:
   - красная шапка,
   - нижний статус (некорректный опыт),
   - протокол ошибок (одна агрегированная запись по zero-run).
3. Создать отдельный опыт с одиночными пропусками -> оранжевая шапка и missing в статусе/протоколе.

## 5) Включение/выключение опыта
1. Отключить опыт (export_on=False).
2. Проверить приглушение в UI и отсутствие опыта в экспортируемом наборе.
3. Включить обратно и повторно экспортировать.

## 6) Common params и qc/fs консистентность
1. Изменить common params в ленте.
2. Проверить, что изменились согласованно:
   - графические шкалы,
   - расчёт qc/fs (таблица/экспорт),
   - статусные значения.

## 7) Проверка сохранения `.zproj`
1. Изменить: common params, видовые флаги (графики/геология/sort/compact), экспорт-on, пару значений qc/fs.
2. Сохранить -> закрыть -> открыть.
3. Убедиться, что state восстановлен без перетирания из старых UI-vars.

## 8) Экспортный контур (GEO / GXL / Excel)
1. Для одного и того же набора опытов выгрузить GEO/GXL/Excel.
2. Сверить:
   - состав опытов,
   - шаг/глубины,
   - калибровочные параметры,
   - qc/fs на 3-5 контрольных строках.
