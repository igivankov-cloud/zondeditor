# GEO export (K2) — логика v7 (рабочая)

## Проблема
1) Экспорт по шаблону падал/резал данные при mismatch rows и при «дотянутых хвостах».
2) GeoExplorer видел отключённые/удалённые опыты, но не видел новые (скопированные), потому что экспорт шёл только по блокам, существующим в шаблоне.

## Решение
Экспорт GEO собирается заново на базе шаблона.

### Что сохраняем из шаблона
- `prefix` — байты до первого блока опыта.
- `22-byte header` каждого блока опыта (marker `FF FF <id> 1E 0A` + метаданные).
- `suffix` — байты после последнего блока (если есть).

### Как формируем выходной GEO
- Берём список `prepared/exported tests` (именно он определяет, какие опыты попадут в файл).
- Для каждого теста:
  - если `tid` есть в шаблоне — используем его header,
  - если `tid` новый — берём базовый header (из первого блока шаблона), подменяем id в marker на новый `tid`.
  - payload строим заново: интерливинг `[qc, fs]` по строкам, длина payload равна фактической длине данных (с выравниванием qc/fs до одного `n`).
- Отключённые/удалённые, которые не входят в `prepared`, в GEO не попадают → GeoExplorer их не показывает.
- Новые (скопированные) появляются в GEO → GeoExplorer их видит.
- «Дотянутые хвосты» сохраняются, т.к. payload может быть любой длины.

## Реализация
Код: `src/zondeditor/io/geo_writer.py` (патч v7).

Ключевые идеи:
- парсинг marker `FF FF <id> 1E 0A`
- карта `headers_by_tid`
- сборка блоков строго по `prepared` в их порядке
- клон header для новых тестов + подмена id
